# basketball-sim

Babylon.js 製の 3D バスケットボール **観戦シミュレーション**。
フルコートで 5 対 5 の AI 同士が自動で試合をする（ユーザー操作は不要）。

既存の `basketball-game` は Havok 物理でボールが暴れて挙動が安定しなかったため、
本プロジェクトは**剛体物理を一切使わず、選手移動とボール軌道をすべてコードで決定論的に制御**している。
これにより毎フレームの結果が予測可能で、暴走しない。

## 起動

```bash
cd C:\Users\user\developsecond\basketball-sim
npm install
npm run dev
```

ブラウザで `http://localhost:5173` を開く。

- 画面上部：スコアボード（チーム名・得点・クォーター・残り時間・ショットクロック）
- 下部ボタン：`1x / 2x / 4x`（再生速度）、`RESTART`
- マウスドラッグでカメラ回転、ホイールでズーム（自動でボールを追従）

## 設計の要点

| 要素 | 方式 |
|---|---|
| ボール | 物理なし。保持・パス・シュート・リバウンドを軌道計算で再現 |
| 選手 AI | 状態機械（オフェンス＝ドライブ/パス/シュート判断、ディフェンス＝マンツーマン＋スティール） |
| シュート成否 | 距離とディフェンス密着度から成功確率を算出し事前抽選 → 軌道アニメーション |
| リバウンド | リム付近の選手から距離・守備側ボーナスで重み付き抽選 |
| クロック | クォーター制（`config.ts` の `QUARTER_TIME` / `QUARTERS`） |
| 速度 | メインループで整数サブステップ実行（dt を大きくせず安定） |

## チューニング

`src/config.ts` に主要パラメータを集約（コート寸法・速度・シュート距離・クロック・チーム色/名）。
AI の挙動は `src/game.ts` の `runOffense` / `runDefense` / `shoot` などを調整する。

## 選手パラメータ（個性）

`src/attributes.ts` に各選手の能力値（0..100）とロスターを定義。フェーズ1では
**走力・シュート決定力（ミドル/3P）・フィニッシュ力・フリースロー・シュート頻度**が
能力値で個別化されている（守備・パス判断・チーム戦術は今後のフェーズで拡張予定）。

```
src/
  config.ts     定数・寸法・チーム定義
  attributes.ts 選手能力値（0..100）とロスター
  util.ts       ベクトル/乱数ヘルパ
  entities.ts   Player / Ball（メッシュ・能力値・論理座標）
  court.ts     コート床・ライン描画・ゴール・ハンドラーリング
  camera.ts    放送風追従カメラ
  game.ts      シミュレーション本体（AI・スコア・クロック・状態機械）
  ui.ts        DOM オーバーレイ（スコアボード・操作）
  main.ts      Babylon エンジン起動とループ配線
```
